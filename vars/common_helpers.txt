<p>
   The common helpers file contains helpers/methods that are to be used across all pipelines.
   You can leverage the helpers/methods as such:
</p>
<p></p>
<code>common_helpers.notifySlackStart()</code>
<p>
   The following is a general list of helpers/methods that are available.
</p>
<dl class="slack-vars">
   <dt>notifySlackSuccess()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Slack in the form of SUCCESS, and accepts two inputs. As of this writing it is expecting <code>channel</code> and an optional <code>generalMessage</code>. <code>channel</code> is the channel name where you want to send the Slack notification. <code>generalMessage</code> is the text that will accompany the SUCCESS notification. If a channel or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifySlackUnstable()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Slack in the form of UNSTABLE, and accepts two inputs. As of this writing it is expecting <code>channel</code> and an optional <code>generalMessage</code>. <code>channel</code> is the channel name where you want to send the Slack notification. <code>generalMessage</code> is the text that will accompany the UNSTABLE notification. If a channel or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifySlackFail()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Slack in the form of FAILED, and accepts four inputs. As of this writing it is expecting <code>channel</code>, <code>message</code>, <code>error</code>, and an optional <code>generalMessage</code>. <code>channel</code> is the channel name where you want to send the Slack notification. <code>message</code> is the error message you want to present to the user (usually defined in the <code>catch</code> statement). <code>error</code> is the system generated error message. <code>generalMessage</code> is additional context to include in the FAILED notification. If a channel or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifySlackStart()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Slack in the form of STARTED, and accepts two inputs. As of this writing it is expecting <code>channel</code> and an optional <code>generalMessage</code>. <code>channel</code> is the channel name where you want to send the Slack notification. <code>generalMessage</code> is the text that will accompany the STARTED notification. If a channel or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifyDiscordSuccess()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Discord in the form of SUCCESS, and accepts two inputs. As of this writing it is expecting <code>discordWebURL</code> and an optional <code>generalMessage</code>. <code>discordWebURL</code> is the webhook URL for the Discord channel you want to send the notification to. <code>generalMessage</code> is the text that will accompany the SUCCESS notification. If a webhook URL or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifyDiscordUnstable()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Discord in the form of UNSTABLE, and accepts two inputs. As of this writing it is expecting <code>discordWebURL</code> and an optional <code>generalMessage</code>. <code>discordWebURL</code> is the webhook URL for the Discord channel you want to send the notification to. <code>generalMessage</code> is the text that will accompany the UNSTABLE notification. If a webhook URL or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifyDiscordFail()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Discord in the form of FAILED, and accepts four inputs. As of this writing it is expecting <code>discordWebURL</code>, <code>message</code>, <code>error</code>, and an optional <code>generalMessage</code>. <code>discordWebURL</code> is the webhook URL for the Discord channel you want to send the notification to. <code>message</code> is the error message you want to present to the user (usually defined in the <code>catch</code> statement). <code>error</code> is the system generated error message. <code>generalMessage</code> is additional context to include in the FAILED notification. If a webhook URL or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifyDiscordStart()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a message to Discord in the form of STARTED, and accepts two inputs. As of this writing it is expecting <code>discordWebURL</code> and an optional <code>generalMessage</code>. <code>discordWebURL</code> is the webhook URL for the Discord channel you want to send the notification to. <code>generalMessage</code> is the text that will accompany the STARTED notification. If a webhook URL or generalMessage is not specified it should default to the values configured in the global settings.</dd>
   <dt>notifyEmailSuccess()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a SUCCESS email notification, and accepts one input. As of this writing it is expecting <code>emailRecp</code>. <code>emailRecp</code> is the recipient address that should receive the email. If an address is not specified it should default to the value configured in the global settings.</dd>
   <dt>notifyEmailFailure()</dt>
   <dd style="margin: 0 0 1em 0">This method sends a FAILURE email notification, and accepts one input. As of this writing it is expecting <code>emailRecp</code>. <code>emailRecp</code> is the recipient address that should receive the email. If an address is not specified it should default to the value configured in the global settings.</dd>
   <dt>catchMe()</dt>
   <dd style="margin: 0 0 1em 0">This method is used to catch exceptions in the pipeline. It accepts five inputs: <code>failureMessage</code>, <code>err</code>, <code>discordWebURL</code>, <code>channel</code>, and <code>emailRecp</code>. <code>failureMessage</code> is the user-friendly description of the error. <code>err</code> is the caught exception. <code>discordWebURL</code>, <code>channel</code>, and <code>emailRecp</code> control where Discord, Slack, and email notifications are sent; each will default to values configured in the global settings when not specified. The method echoes the error, marks the build as failed, sends notifications through Slack, Discord, and email, and rethrows the error.</dd>
   <dt>catchMeNoDiscord()</dt>
   <dd style="margin: 0 0 1em 0">This method is used to catch exceptions in the pipeline when Discord notifications are not desired. It accepts four inputs: <code>failureMessage</code>, <code>err</code>, <code>channel</code>, and <code>emailRecp</code>. <code>failureMessage</code> is the user-friendly description of the error. <code>err</code> is the caught exception. <code>channel</code> and <code>emailRecp</code> control where Slack and email notifications are sent; each will default to values configured in the global settings when not specified. The method echoes the error, marks the build as failed, sends notifications through Slack and email, and rethrows the error.</dd>
   <dt>catchMeNoDisOrSlack()</dt>
   <dd style="margin: 0 0 1em 0">This method is used to catch exceptions in the pipeline when neither Slack nor Discord notifications are needed. It accepts three inputs: <code>failureMessage</code>, <code>err</code>, and <code>emailRecp</code>. <code>failureMessage</code> is the user-friendly description of the error. <code>err</code> is the caught exception. <code>emailRecp</code> controls where email notifications are sent and will default to the value configured in the global settings when not specified. The method echoes the error, marks the build as failed, sends the email notification, and rethrows the error.</dd>
   <dt>noUpdates()</dt>
   <dd style="margin: 0 0 1em 0">This method communicates that a build completed without performing any updates. It accepts five inputs: <code>extraMessage</code>, <code>discordWebURL</code>, <code>channel</code>, <code>emailRecp</code>, and <code>generalMessage</code>. <code>extraMessage</code> provides context about why no updates were needed. The remaining parameters control where notifications are sent and default to the values configured in the global settings. The method sends an email noting that no updates were conducted, posts an ABORTED result to Discord, and sends a NO UPDATES message to Slack.</dd>
</dl>
